# al-maktaba — technical specification

## overview

al-maktaba — современный веб-интерфейс для арабской библиотеки Maktabah Shamela. цель проекта — дать пользователям комфортное чтение классических арабских книг через адаптивный, быстрый и минималистичный UI. проект копирует контент shamela.ws (~8 425 книг, ~7 млн страниц, ~3 146 авторов, 40 категорий), но полностью переосмысливает взаимодействие с ним.

основная аудитория — студент или исследователь исламских наук, читающий арабский, без доступа к десктопному приложению Shamela или предпочитающий web. язык интерфейса переключается (ru/en, по умолчанию ru); сами книги остаются на арабском, ничего не переводится. в будущем планируется добавить больше языков интерфейса.

## user stories

- как читатель, я захожу на главную и за 5 секунд понимаю как найти книгу — через поиск или через каталог категорий
- как читатель, я могу открыть книгу и читать её страницы с комфортным шрифтом, настраивая размер и огласовки под себя
- как читатель, я могу поставить закладку на страницу одним кликом и вернуться к ней из /me
- как читатель, я могу выделить фрагмент текста и сразу скопировать/сохранить в заметки/поделиться
- как читатель на телефоне, я могу листать страницы свайпом и видеть прогресс чтения
- как читатель, я вижу блок "продолжить чтение" на главной и в один клик возвращаюсь к последней книге
- как пользователь двух устройств, я могу экспортировать свои закладки/заметки в JSON и импортировать на другом устройстве
- как англоязычный пользователь, я переключаю интерфейс на английский одним кликом в хедере — весь UI становится на английском, данные книг остаются арабскими

## features (v0.1)

### 1. главная страница

- hero секция: краткое описание проекта + кнопки "каталог" и "поиск"
- блок **"продолжить чтение"** (только если у пользователя есть история) — карточки последних 3-5 книг с прогрессом
- секция **"категории"** — 8-10 мета-групп (арабские названия); под каждой — счётчик книг
- секция **"недавно добавленные"** — последние 12-20 книг из базы
- футер с упоминанием источника: `البيانات من المكتبة الشاملة`
- **acceptance:** главная рендерится ≤500ms (SSR), lighthouse score ≥90 performance

### 2. каталог категорий (`/categories`)

- 40 оригинальных категорий Shamela сгруппированы в 8-10 мета-групп _(group structure → claude's pick)_:
  - `العقيدة والفرق` — акыда + фирак (1, 2)
  - `القرآن وعلومه` — тафсир + улум + таджвид (3, 4, 5)
  - `الحديث وعلومه` — кутуб-ас-сунна + шурух + тахридж + улум аль-хадис + иляль (6, 7, 8, 9, 10)
  - `الفقه وأصوله` — усуль + фикх 4 мазхабов + общий + фатва (11-22)
  - `الرقائق والسيرة والتراجم` — рикак + сира + тарих + тараджим (23-26)
  - `اللغة والأدب` — язык + гариб + нахв + адаб + балага + шеир (27, 29-35)
  - `البلدان والأنساب` — ансаб + булдан (27, 28)
  - `الجوامع وعام` — джавами + фихарис + тыбб + общее + другое (36-40)
- клик на мета-группу раскрывает подкатегории
- **acceptance:** пользователь за 2 клика доходит от главной до списка книг любой категории

### 3. страница категории (`/categories/[id]`)

- заголовок (арабский), breadcrumbs (язык интерфейса + арабский название категории)
- счётчик книг + поиск по названиям внутри категории
- **пагинация** — по 30 книг на страницу, URL с параметром `?page=N`
- **сортировка** (в URL `?sort=...`): по году смерти автора (default), по алфавиту (ا...ي), по количеству страниц
- **режим отображения** (toggle): карточки / компактный список
- каждая карточка: название (ар), автор (ар), год смерти (хиджра + милади), количество страниц
- **acceptance:** смена сортировки не перезагружает страницу (клиентский роутинг), URL обновляется, back работает корректно

### 4. индекс авторов (`/authors`)

- 3 вкладки сортировки: `по году смерти` (default), `по алфавиту`, `по количеству книг`
- поле автодополнения (autocomplete) по именам
- список: имя автора (ар), годы жизни (хиджра/милади), количество книг
- **acceptance:** поиск находит автора по первым 2-3 символам имени за ≤100ms

### 5. страница автора (`/authors/[id]`)

- имя автора (h1, ар), годы жизни (хиджра + милади)
- список всех книг автора (тот же формат что в категории, пагинация, сортировка)
- **acceptance:** все книги автора загружаются одним запросом за ≤200ms

### 6. карточка книги (`/books/[id]`)

- название книги (h1, ар), автор (ссылка), категория (ссылка на мета-группу)
- обложка из `cover.db` (если есть)
- метаданные блоком: издательство, редактор, издание, год, количество страниц, количество частей
- кнопка **"начать чтение"** → `/books/[id]/pages/1`
- кнопка **"в избранное"** (toggle)
- блок **оглавление книги** — дерево глав из таблицы `titles`, клик по главе → соответствующая страница
- **acceptance:** карточка со всеми метаданными и оглавлением рендерится ≤400ms

### 7. чтение страницы (`/books/[id]/pages/[page]`)

- **макет:** центрированный текст + левая боковая панель с оглавлением (сворачивается)
- **текст страницы:** арабский RTL, шрифт Amiri по умолчанию
- **прогресс-бар сверху** — видно сколько прочитано (`N / total`)
- **навигация:**
  - кнопки `<<` `<` [input] `>` `>>` (первая, назад, номер, вперёд, последняя)
  - горячие клавиши: ← / → (или J/K)
  - свайп влево/вправо на мобильных
- **панель инструментов:** размер шрифта (-A/+A), огласовки On/Off, тема "сепия", поделиться, поиск в книге, скачать PDF (если есть)
- **выделение текста:** при выделении показывается мини-меню — "копировать цитату", "добавить заметку", "поделиться ссылкой на фрагмент"
- **закладка** — иконка закладки в правом верхнем углу страницы (toggle)
- оглавление в сайдбаре подсвечивает текущую главу
- **acceptance:**
  - страница (7 млн записей в БД) рендерится ≤300ms серверно
  - навигация между страницами (prefetch следующей) — мгновенная
  - огласовки On/Off — клиентская фильтрация без перезагрузки

### 8. поиск (`/search`)

- хедер: моментальный мини-поиск (autocomplete) по названиям книг и авторов
- страница `/search`: 3 вкладки — `книги`, `авторы`, ~~`содержимое`~~ (full-text — в v0.2)
- результаты: карточки с подсветкой совпадений, пагинация
- URL с query: `/search?q=...&type=books`
- **acceptance:** первые 10 результатов за ≤150ms (Postgres ilike + индексы)

### 9. персональные функции (без авторизации)

все личные данные привязаны к `session_id` (cookie). **claude's pick** — экспорт/импорт JSON для переноса между устройствами.

- `/bookmarks` — список закладок (книга + страница + timestamp + контекст)
- `/notes` — список заметок (книга + страница + текст + timestamp)
- `/history` — история чтения (книга + последняя страница + прогресс %)
- `/favorites` — избранные книги (список)
- `/me` — дашборд: последние закладки (5), последние заметки (5), "продолжить чтение" (3), избранное (6)
- `/settings` → **экспорт JSON** (одна кнопка = файл со всеми личными данными) и **импорт** (загрузить JSON → перезаписать/слить)
- **acceptance:** любую личную функцию доступна с карточки книги/страницы чтения, не нужно идти в отдельное меню

### 10. локализация UI (ru/en)

- все UI-тексты (кнопки, подписи, сообщения, breadcrumbs, формы, ошибки) — в словаре i18n (`locales/ru.json`, `locales/en.json`)
- переключатель языка в хедере (флаги или `ru/en` текстом)
- URL-префикс `/ru/...` и `/en/...` (next-intl)
- `hreflang` теги на всех страницах для SEO
- автодетект по `Accept-Language` при первом визите, потом сохраняется в cookie
- **данные книг НЕ переводятся** — категории, имена авторов, названия книг, текст всегда арабские
- **acceptance:** вся навигация, кнопки, плейсхолдеры, тексты ошибок переключаются при смене языка; арабский контент остаётся неизменным

### 11. темы (light / dark / sepia)

- 2 темы сайта: **light** (по умолчанию) и **dark** (переключатель)
- **sepia** — режим читалки (только на странице чтения): тёплый бежевый фон, коричневатый текст
- переключатели — в хедере (light/dark) и в панели читалки (sepia)
- реализация через CSS переменные (`--bg`, `--ink`, `--accent`, ...), сохранение выбора в localStorage
- **acceptance:** переключение темы без мерцания (flash of wrong theme), сохраняется между сессиями

### 12. PWA (установка как приложение)

- манифест `manifest.json`, иконки 192/512, theme_color, background_color
- service worker для кэширования статики (не страниц книг)
- "добавить на главный экран" работает на iOS и Android
- **acceptance:** установленное приложение открывается на весь экран без браузерной обвязки; статика работает без сети (страницы книг требуют сеть)

## data model

текущая схема в `backend/src/database/schema.ts` — 10 таблиц postgres:

```
categories       id, name (ar), order, book_count
authors          id, name (ar), biography, death_number (hijri int), death_text
books            id, name (ar), author_ids[], category_id, bibliography, date,
                 hint, metadata (json), pdf_links (json), printed, type,
                 major_release, minor_release, pages_count
book_authors     book_id, author_id                              (M:M)
pages            id, book_id, shamela_id, content (html+ar), part, page, number
titles           id, book_id, shamela_id, content (ar), page_shamela_id, parent_shamela_id
bookmarks        id, session_id, book_id, page_id, created_at
notes            id, session_id, book_id, page_id, content, created_at, updated_at
reading_history  id, session_id, book_id, page_id, progress, updated_at
favorites        id, session_id, book_id, created_at
```

**индексы (критичные для производительности):**
- `pages (book_id, shamela_id)` — постраничная навигация
- `pages (book_id)` — все страницы книги
- `books (category_id)` — книги категории
- `books` trigram index по `name` для поиска
- `authors` trigram index по `name`
- `bookmarks/notes/history/favorites (session_id)` — личные данные
- `titles (book_id, parent_shamela_id)` — дерево оглавления

**особенности v4:**
- текст книг хранится в Lucene индексе (`store/page/`), не в SQLite. импорт извлекает текст через Java → TSV → `COPY pages FROM ...`
- метаданные страниц берутся из `book/NNN/{id}.db` (поля `part`, `page`, `number`) и мержатся с текстом по `shamela_id`

## api / routes

```
# категории
GET  /api/categories                          → список 40 категорий с мета-группами
GET  /api/categories/:id                      → детали + книги (page, sort)
GET  /api/categories/:id/books?page=&sort=    → пагинация книг

# авторы
GET  /api/authors?sort=death|alpha|books&q=   → список с сортировкой + поиск
GET  /api/authors/:id                         → автор + все его книги
GET  /api/authors/autocomplete?q=             → ≤10 совпадений для автокомплита

# книги
GET  /api/books/:id                           → карточка + оглавление
GET  /api/books/:id/titles                    → дерево оглавления
GET  /api/books/:id/pages/:page               → содержимое страницы
GET  /api/books/autocomplete?q=               → для хедера поиска
POST /api/books/:id/search?q=                 → поиск внутри книги (ilike по pages.content)

# поиск
GET  /api/search?q=&type=books|authors        → глобальный

# личные функции (session_id в cookie)
GET  /api/bookmarks                           → все закладки
POST /api/bookmarks/toggle                    → { bookId, pageId }
GET  /api/notes
POST /api/notes                               → create/update
DEL  /api/notes/:id
GET  /api/history
POST /api/history                             → upsert { bookId, pageId, progress }
GET  /api/history/books/:bookId               → прогресс конкретной книги
GET  /api/favorites
POST /api/favorites/toggle                    → { bookId }

# экспорт/импорт личных данных
GET  /api/me/export                           → JSON со всеми личными данными
POST /api/me/import                           → загрузить JSON
```

все ответы — `application/json`. swagger на `/api/docs`.

## technical stack

**backend (уже заложено):**
- **node.js 20, typescript strict**
- **nestjs 11** — модульная архитектура
- **drizzle orm** с драйвером `postgres-js` (не `node-postgres`)
- **postgresql 16** (docker compose)
- **class-validator + class-transformer** для DTO
- **better-sqlite3** для импорта из Shamela .db
- **java 21 + lucene 9.5** для извлечения текста из store/page/

**frontend (ещё не создан, планируется):**
- **next.js 15** app router, server components + streaming
- **tailwind css 4** c CSS переменными для тем
- **shadcn/ui** для базовых компонентов (button, dialog, dropdown, input)
- **next-intl** для локализации ru/en
- **tanstack query** для клиентского кеша данных
- **nuqs** для URL-state (пагинация, фильтры)
- **tailwindcss-rtl** или logical properties для RTL контента

**арабский текст:**
- шрифт **Amiri** (Google Fonts) для всего арабского контента
- `dir="rtl"` только на контейнере арабского текста, не на `<html>`
- UI остаётся `dir="ltr"`, текст книг — `dir="rtl"` внутри

**инфраструктура:**
_решение отложено — в SPEC фиксируются только базовые принципы:_
- SSR-приложение на node, postgres отдельно
- CDN для статики (next.js build output)
- конкретный VPS/домен/аналитика — решается при подготовке к запуску

## ui / ux

**общий тон:** спокойная классика — как хорошая книга. светлый бежеватый фон, чёрный текст, тёплый золотистый (охра) акцент. никакой "гипердизайн-студии" — минимум теней, скруглений, анимаций.

**ключевые страницы:**

1. **главная** — hero с двумя кнопками, блок "продолжить чтение" (если есть), 8-10 больших карточек мета-категорий, горизонтальный скролл "недавно добавленные"
2. **каталог категории** — заголовок + фильтры/сортировка, пагинация, компактные карточки книг
3. **карточка книги** — обложка слева, метаданные справа, оглавление под ними
4. **читалка** — центрированный текст (max-width 720px), левый сайдбар с оглавлением (сворачивается), верхний прогресс-бар, инструменты в плавающей панели
5. **поиск** — одна строка + табы + результаты с подсветкой
6. **/me** — 4 блока: закладки, заметки, продолжить чтение, избранное

**типографика:**
- UI (RU/EN): системный sans-serif (-apple-system, Segoe UI, Roboto)
- арабский текст: **Amiri** (serif), `font-size` 18-22px для чтения, line-height 1.9 для RTL
- заголовки UI — чуть крупнее, тот же sans-serif, слегка увеличенный weight

**палитра (light):**
- `--bg: #fafaf7` (тёплый бежеватый)
- `--ink: #1a1a1a`
- `--accent: #c9a24d` (золотистый)
- `--line: #e5e5e0`

**палитра (dark):**
- `--bg: #14161a`
- `--ink: #eee`
- `--accent: #e6b966`
- `--line: #2a2f38`

**палитра (sepia — только читалка):**
- `--bg: #f4ecd8`
- `--ink: #5b4636`
- тёплый коричневатый оттенок, меньше утомляет глаза

**интерактивные элементы:**
- все тап-таргеты на мобильных ≥44px
- фокус-стили видимы (outline accent)
- reduced-motion поддерживается — анимации отключаются
- RTL тексты — `dir="rtl"` на `.arabic-content`

## non-functional requirements

- **performance:**
  - главная (SSR): TTFB ≤300ms, LCP ≤1s
  - страница чтения: серверный рендер ≤300ms, prefetch следующей страницы
  - автодополнение поиска: ≤150ms от запроса до результата
  - lighthouse performance ≥90 на 3G mobile
- **мобильность:** mobile-first, iOS Safari + Android Chrome — первичный таргет
- **PWA:** манифест + иконки, установка на главный экран, оффлайн-UI (но не содержимое)
- **accessibility:**
  - WCAG 2.1 AA: контраст текста ≥4.5:1, интерактивные ≥3:1
  - клавиатурная навигация везде
  - ARIA labels на иконках без текста
  - screen reader корректно читает арабский (lang="ar" на контенте)
  - respects `prefers-reduced-motion` и `prefers-color-scheme`
- **SEO:**
  - мета-теги title/description на всех страницах (шаблоны под языки)
  - `sitemap.xml` со всеми 8 425 книгами и основными страницами
  - `robots.txt` разрешает индексацию
  - `hreflang` для RU/EN дубликатов URL
  - OpenGraph для шаринга в соцсети (позже)
- **i18n:** полная локализация UI (ru/en), по умолчанию ru; переключатель в хедере; словари в `locales/`
- **privacy:** без аналитики на основе cookies; Plausible/Umami (решение позже); session_id — единственная cookie
- **scale:** v0.1 — до 5 000 DAU; архитектура позволяет горизонтальное масштабирование Next.js, но в v0.1 это не нужно

## out of scope (v0.1)

- **полнотекстовый поиск по содержимому книг** — требует Elasticsearch/Meilisearch + индексацию 7 млн страниц; отложено до v0.2. в v0.1 — только поиск по названиям и авторам (postgres ilike + trigram)
- **оффлайн чтение** — сложная синхронизация, редко нужно; PWA-оболочка есть, но контент страниц требует сеть
- **синхронизация между устройствами через сервер** — в v0.1 только экспорт/импорт JSON. код-синхронизация и email-link — в v0.2
- **авторизация** — в v0.1 всё по session_id. логин через OAuth/email — отдельный проект v0.3+
- **комментарии / обсуждения** — не соответствуют академическому характеру сервиса
- **словарь исламских терминов** — большая работа, отложено до v0.2
- **дополнительные языки UI** — в v0.1 только ru/en. архитектура i18n легко добавит другие позже
- **мобильные нативные приложения** — PWA покрывает 80% потребности в v0.1
- **общение с API Shamela** — ждём API ключ; v0.1 работает с локально импортированной базой из ISO Shamela v4

## open questions

- **группировка 40 категорий в 8-10 мета-групп** — в SPEC предложен claude's pick, но финальные названия и группировка могут уточниться когда увижу реальные категории
- **обложки книг (cover.db)** — формат и качество неизвестны. если плохое качество — не показывать вообще
- **PDF-линки** — в некоторых книгах есть `pdf_links` на внешние источники (archive.org). показывать ли кнопку "скачать PDF" или это отвлекает от чтения?
- **удаление личных данных** — в v0.1 cookie session_id живёт вечно. нужен ли "стереть всё" + автоудаление через 6 месяцев неактивности?
- **сепия vs dark для читалки** — сепия работает только на светлом фоне. если пользователь в dark mode и включает сепию на читалке — как быть? (предварительное решение: сепия переопределяет dark на странице чтения)
- **данные** — основной блокер: без полного Lucene индекса чтения не будет. две траектории:
  - **сценарий A** (ISO): получаем полную базу от брата/знакомого с Windows-установкой, извлекаем текст через Java, импортируем. работаем с зафиксированной версией 1446.1
  - **сценарий B** (API): получаем ключ от `mail@shamela.ws`, используем npm пакет `shamela` или прямые HTTP-вызовы, данные свежие. v0.1 закладывает архитектуру под любой сценарий: импорт — отдельный модуль, основная БД от него не зависит
